# Hotfix Blob

**Instant fixes. Zero downtime. No deploys.**

Push critical configuration changes to your entire infrastructure in seconds without redeployment or service interruption.

---

## What You Get

### Instant Emergency Updates

When something breaks, fix it immediately:

**Sub-Second Delivery**: Push config changes to all clients instantly

**Zero Downtime**: No restarts or redeployments needed

**Atomic Updates**: Changes apply completely or not at all

**One-Click Rollback**: Revert to previous state instantly

### Common Use Cases

**Provider Emergency Disable:**
```json
{
  "action": "disable_provider",
  "provider": "openai/gpt-4",
  "reason": "Upstream API degradation",
  "expires_in": "2h"
}
```

**Rate Limit Override:**
```json
{
  "action": "update_rate_limit",
  "tenant_id": "tenant_abc",
  "new_limit": 1000,
  "reason": "Emergency capacity increase"
}
```

**Circuit Breaker Override:**
```json
{
  "action": "force_circuit_open",
  "provider": "anthropic/claude-3",
  "reason": "Detected quality degradation",
  "duration": "30m"
}
```

## How It Works

<details>
<summary><strong>1. Create Hotfix</strong></summary>

From dashboard or CLI:

```bash
# Via CLI
igris hotfix create \
  --action disable_provider \
  --provider openai/gpt-4 \
  --reason "API degradation" \
  --expires 2h
```

```bash
# Via API
POST /v1/hotfixes
{
  "action": "disable_provider",
  "provider": "openai/gpt-4",
  "expires_in": "2h"
}
```

</details>

<details>
<summary><strong>2. Instant Distribution</strong></summary>

Hotfix distributes to all clients:

- **Ed25519 Signature**: Cryptographically signed for security
- **Pull Refresh**: Periodic polling (60s interval)
- **Version Control**: Monotonically increasing version numbers
- **Auto-Expiration**: Policies expire automatically

</details>

<details>
<summary><strong>3. Automatic Validation</strong></summary>

Before applying:

- **Signature Verification**: Cryptographic validation
- **Schema Check**: Ensure valid configuration
- **Safety Gates**: Prevent dangerous changes
- **Conflict Detection**: Check for incompatible updates

</details>

<details>
<summary><strong>4. Atomic Application</strong></summary>

Changes apply all-or-nothing:

- **No Partial States**: Either fully applied or fully rolled back
- **Health Checks**: Monitor impact immediately
- **Auto-Rollback**: Revert if errors increase

</details>

## Security

### Cryptographic Signing

- **Algorithm**: Ed25519 signatures
- **Verification**: All clients verify before applying
- **Key Rotation**: Regular automatic rotation
- **Audit Trail**: Complete history of all hotfixes

### Multi-Signature Approval

For production environments:

- **Require 2+ Approvals**: Multiple admins must approve
- **Time-Locked**: Enforced delay before application
- **Emergency Override**: Break-glass for critical incidents

## API Reference

### Create Hotfix

```bash
POST /v1/hotfixes
{
  "action": "disable_provider",
  "target": "openai/gpt-4",
  "reason": "API degradation detected",
  "expires_in": "2h",
  "rollout_strategy": "gradual"
}
```

### List Active Hotfixes

```bash
GET /v1/hotfixes?status=active
```

### Rollback Hotfix

```bash
POST /v1/hotfixes/{hotfix_id}/rollback
{
  "reason": "Issue resolved"
}
```
